OCI Observability API Operations and Permissions

This page details the specific API operations for OCI Observability services and the permissions required for each operation. Use this information to create precise OCI policies when migrating from AWS.

Observability API Operations Reference

In OCI, observability operations require specific permission strings for monitoring, logging, and other telemetry services. This reference helps you map AWS CloudWatch, CloudTrail, and X-Ray permissions to equivalent OCI observability permissions.

Source Documentation: This information is based on the OCI Monitoring Policy Reference, OCI Logging Policy Reference, and OCI APM Permissions Reference.
OCI Observability Resource Types

OCI uses several resource types to manage observability resources:

  • alarm - Monitoring alerts based on metric conditions
  • metrics - Time-series data collected from resources
  • log-group - A container for organizing logs
  • log - Log data collected from resources
  • apm-domain - Container for APM traces and monitoring
  • apm-synthetics - Synthetic monitoring for applications
  • logging-analytics - Advanced log analytics and processing
  • logging-analytics-entity - Resources managed by logging analytics
  • logging-analytics-log-group - Log groups for logging analytics
  • logging-analytics-source - Sources of logs for logging analytics
  • logging-analytics-namespace - Organizational unit for logging analytics

Creating AWS-to-OCI Observability Policies
Example: AWS CloudWatch Administrator Policy
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "cloudwatch:*"
      ],
      "Resource": "*"
    }
  ]
}
Equivalent OCI Policy:
Allow group MonitoringAdmins to manage metrics in compartment MyCompartment
Allow group MonitoringAdmins to manage alarms in compartment MyCompartment
Example: AWS CloudWatch Logs Reader Policy
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "logs:Describe*",
        "logs:Get*",
        "logs:List*",
        "logs:StartQuery",
        "logs:StopQuery",
        "logs:TestMetricFilter",
        "logs:FilterLogEvents"
      ],
      "Resource": "*"
    }
  ]
}
Equivalent OCI Policy:
Allow group LogReaders to inspect log-groups in compartment MyCompartment
Allow group LogReaders to inspect logs in compartment MyCompartment
Allow group LogReaders to read log-content in compartment MyCompartment
Example: AWS X-Ray Full Access Policy
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "xray:*"
      ],
      "Resource": "*"
    }
  ]
}
Equivalent OCI Policy:
Allow group ApmAdmins to manage apm-domains in compartment MyCompartment
Allow group ApmAdmins to manage apm-synthetics in compartment MyCompartment
Understanding OCI Observability Permission Strings
OCI Permission String Description Used For
METRIC_INSPECT View metric metadata List and describe metrics
METRIC_READ Read metric data Query and visualize metrics
METRIC_EMIT Publish metric data Send custom metrics
ALARM_INSPECT View alarm metadata List and describe alarms
ALARM_CREATE Create alarms Create new alarms
ALARM_UPDATE Modify alarms Update alarm configurations
ALARM_DELETE Delete alarms Remove alarms
LOG_GROUP_INSPECT View log group metadata List and describe log groups
LOG_GROUP_CREATE Create log groups Create new log groups
LOG_GROUP_UPDATE Modify log groups Update log group configurations
LOG_GROUP_DELETE Delete log groups Remove log groups
LOG_INSPECT View log metadata List and describe logs
LOG_READ Read log content View log entries
LOG_CREATE Create logs Create new logs
LOG_UPDATE Modify logs Update log configurations
LOG_DELETE Delete logs Remove logs
