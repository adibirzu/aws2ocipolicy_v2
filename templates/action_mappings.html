AWS to OCI Action Mappings

This page provides a comprehensive mapping between AWS actions and their Oracle Cloud Infrastructure (OCI) equivalents, based on the official Oracle documentation.

AWS Action Pattern OCI Verb OCI Resource Description
*:List* inspect {service} View resources in a service
*:Get* read {service} View detailed information about a resource
*:Describe* inspect {service} View resources and their details
*:Create* manage {service} Create new resources
*:Delete* manage {service} Delete resources
*:Update* manage {service} Update/modify resources
*:Put* manage {service} Add/update resources or their attributes
*:Start* use {service} Start/launch resources
*:Stop* use {service} Stop/shutdown resources
*:Enable* manage {service} Enable features or services
*:Disable* manage {service} Disable features or services
*:* manage all-resources Wildcard for all actions (administrative access)
OCI Verbs & Their Meanings
OCI Verb Description AWS Equivalent
inspect View resources but not their detailed contents (list resources) List*, Describe*
read View detailed information about resources (see contents) Get*
use Use resources without modifying them (execute, run) Start*, Stop*, Invoke*
manage Create, update, delete resources (full control) Create*, Delete*, Update*, Put*

When translating from AWS actions to OCI, these basic verbs are combined with resource types to create OCI policy statements.

For example, s3:GetObject becomes Allow group [group] to read object-storage objects in tenancy
