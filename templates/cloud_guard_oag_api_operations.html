OCI Cloud Guard and Oracle Access Governance API Operations

This page details the specific API operations for OCI Cloud Guard and Oracle Access Governance (OAG) services and the permissions required for each operation. Use this information to create precise OCI policies when migrating from AWS.

Cloud Guard and OAG API Operations Reference

Cloud Guard and Oracle Access Governance are OCI-specific security services that provide continuous monitoring, detection, and remediation capabilities, as well as identity governance. These services don't have direct equivalents in AWS but provide similar functionality to AWS Security Hub, AWS Config, and AWS IAM Access Analyzer.

Source Documentation: This information is based on the OCI Cloud Guard Policy Reference and Oracle Access Governance Policy Reference.
OCI Cloud Guard and OAG Resource Types

OCI uses several resource types to manage Cloud Guard and OAG resources:

  • cloud-guard-family - An aggregate resource-type covering all Cloud Guard resources
  • cloud-guard-target - Scope of resources monitored by Cloud Guard
  • cloud-guard-detector-recipe - Configuration for Cloud Guard's detection capabilities
  • cloud-guard-responder-recipe - Configuration for Cloud Guard's automated responses
  • cloud-guard-problem - Security issue detected by Cloud Guard
  • access-governance-instance - Oracle Access Governance service instance
  • governance-instance - Alternative name for access governance instance
  • governance-report - Access reports generated by Oracle Access Governance
  • governance-campaign - Access review campaigns in Oracle Access Governance

Creating Policies for Cloud Guard and OAG
Example: Cloud Guard Administrator Policy
Allow group CloudGuardAdmins to manage cloud-guard-family in tenancy
Allow group CloudGuardAdmins to read compartments in tenancy
Allow group CloudGuardAdmins to use tag-namespaces in tenancy
Example: Cloud Guard Analyst Policy
Allow group CloudGuardAnalysts to read cloud-guard-family in compartment MyCompartment
Allow group CloudGuardAnalysts to inspect cloud-guard-problems in compartment MyCompartment
Example: Oracle Access Governance Administrator Policy
Allow group OAGAdmins to manage governance-instances in compartment MyCompartment
Allow group OAGAdmins to manage governance-campaigns in compartment MyCompartment
Allow group OAGAdmins to manage governance-reports in compartment MyCompartment
Example: Oracle Access Governance User Policy
Allow group OAGUsers to read governance-instances in compartment MyCompartment
Allow group OAGUsers to read governance-campaigns in compartment MyCompartment
Allow group OAGUsers to read governance-reports in compartment MyCompartment
AWS to OCI Migration Note: Cloud Guard and Oracle Access Governance don't have direct equivalents in AWS, but they provide similar functionality to AWS Security Hub, AWS Config, and AWS IAM Access Analyzer. When migrating from AWS to OCI, you'll need to create new policies specifically for these services based on your security requirements.
